モデルファイルにリレーション用のメソッドを定義する事により、関連する別テーブルのデータを容易に取得することができるようになります。ここでは以下の状況を考えてみます。

- ユーザーは、多数のブログ投稿を持っている（User has many posts）
- ブログ投稿は、ユーザーに所属する（Post belongsTo user）

以降では、「Factory と Faker」で見た Post モデルやマイグレーション等が、準備済みと想定しています。

この時、リレーション用のメソッドは以下のように定義できます。（名前規則に従う書き方）
（Post の外部キーも名前規則に従って「user_id」となっている前提です）

```php
class User extends Authenticatable
{
    public function posts()
    {
        return $this->hasMany(Post::class);
    }
}

class Post extends Model
{
    public function user()
    {
        return $this->belongsTo(User::class);
    }
}
```


以下、補足です。
```php
// 補足1
// Xxx::class の形式でクラスを指定した場合、名前空間の import は不要です。同じ階層にあるファイルだからです。

// 補足2
// orderBy() などを繋げることも可能です。
public function posts()
{
    return $this->hasMany(Post::class)->orderBy('id');
}
```
