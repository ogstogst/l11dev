ルートモデル結合機能を使うことにより、URL の連番などから、Laravel が自動で対応するモデルを取得してくれるようになります。以下サンプルです。

web.php
```php
use App\Http\Controllers\HelloController;

Route::get('user/{user}', [HelloController::class, 'show']);
```

HelloController.php
```php
use App\Models\User;

class HelloController extends Controller
{
    public function show(User $user)
    {
        dd($user->toArray());
    }
}
```

ルート定義で使用したパラメータ名 `user` と、コントローラ側メソッドに指定するパラメータ名 `user` を合わせた上で、モデルをタイプヒントする事により、ルートモデル結合機能が利用できます。Laravel では裏側で、以下のようなコードが実行されています。

```php
User::findOrFail(連番);
```

該当データが無い時は、`404 Not Found` 画面が表示されます。

上記は、「暗黙の結合」という方法になります。他に「明示的な結合」と言い、自分でデータ取得ロジックを書く方法もありますが、ここでは割愛させていただきます。
